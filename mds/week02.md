#### 프리미티브 타입 종류와 값의 범위 그리고 기본 값
- - -

- Primitive Type (기본형, 프리미티브 타입)
	
	> 8가지 기본형 타입이 있습니다.
	1. Integer
		- byte 
			- 기본값 0
			- 값의 범위 : **-128 ~ 127** // 1바이트. 8비트 표현 범위입니다.
		- short
			- 기본값 0
			- 값의 범위 : **-32,768 ~ 32,767** // 2바이트, 16비트 내에서 표현할 수 있습니다.
		- int (default)
			- 기본값 0
			- 값의 범위 : **-2,147,483,648 ~ 2,147,483,647** // 4바이트 내에서 표현가능해요.
		- long
			- 기본값 0
			- 값의 범위 : **-9,223,372,036,854,775,808 ~ 9,223,372,036,854,775,807** //8바이트
	2. Floating-Point
		- float
			- 기본값 0.0
			- 값의 범위 : 소수점 7 자리까지 // 4바이트
		- double (defualt)
			- 기본값 0.0
			- 값의 범위 : 소수점 16 자리까지 // 8바이트 내에서 표현가능해요.		
	3. char
			- 기본값 '\u0000'
			- 값의 범위 : **0 ~ 65,535** // 2바이트 (유니코드)
	4. boolean
			- 기본값 false
			- 값의 범위 : **true, false** // 1바이트

#### 프리미티브 타입과 레퍼런스 타입
- - -
- 프리미티브 타입은 위에 작성한 내용처럼 자바에서 미리 정의한 자료형입니다. 
- 많이 사용하는 자료형을 미리 정의한 것으로 대부분 다른 언어에서도 공통으로 사용하는 자료형들입니다. 
- 레퍼런스 타입은 사용자가 정의해서 사용하는 자료형입니다.
- 프리미티브 타입은 기본값이 있지만, 레퍼런스 타입은 null이 기본값 일 수 있습니다.
- 프리미티브 타입의 크기는 데이터 유형에 따라 다르지만, 레퍼런스 타입은 동일한 크기입니다.

#### 리터럴
- - -
- 고정시키는 값, 숫자형, 문자형 등이 있습니다. 
- int num = 100; //100이 리터럴입니다. 
- 많은 곳에서 사용하는 값이라면 리터럴을 별도로 정의해서 사용하면 나중에 수정하기에 좋습니다. num = 100 보다는 별도로 정의한 100 이라는 값을 정의한 문자로 대체. 예. int num = HUNDRED; HUNDRED에 100을 별도 정의한 상태

#### 변수 선언 및 초기화하는 방법
- - -
- 자료형과 변수명을 적어주는 것을 선언이라고 합니다. 예. int num;
- 선언한 변수에 값을 대입하여 의미있는 값을 할당하는 것을 초기화라고합니다.예. num = 123;

- 변수
	- 지역변수 (Local Variable)
		- 메서드 내부에 선언합니다.
		- 스코프
			-메서드 안에서만 사용가능합니다.
			- 해당 메서드 밖에서는 사용 불가능합니다.
			- 다른 곳에 같은 이름의 변수가 있어도 연동하지 않고 별도로 인식합니다. 해당 메서드 내에서만 사용할 수 있기 때문입니다.
		- 라이프타임
			- stack 메모리에 생성하고, 메서드 종료시 사라집니다.
	- 인스턴스 변수 (Instance Variable)
		- heap메모리에 생성합니다. stack 보다 용량 한계가 적습니다.
		- 인스턴스를 생성하면 사용할 수 있습니다.
		- 클래스 변수와 달리 인스턴스 생성하면 새로운 저장공간을 할당하여 공유하지 않습니다.
		- 스코프
			- public이면 동일한 패키지내 다른 클래스에서 사용할 수 있습니다.
		- 라이프타임
				- 개체가 메모리에 유지될 때까지 입니다.
	- static 변수 (Class Variable)
		- 클래스 내에서 static을 붙여 선언.
		- 자바에서는 지양해야 함. 코드가 많아지면 추적이 어려움.출처: http://me2.do/5sEMN45r
		- 스코프
			- 프로그램이 종료전까지 사용할 수 있다.
			- 자주 사용할 값을 static 변수로 사용하면 효율적이다.
		- 라이프타임
			- 프로그램 종료 전까지입니다.
#### 타입 변환, 캐스팅 그리고 타입 프로모션
- - -
- 자료형 변환하는 것을 타입 변환이라고 해요.
- 작은 데이터와 큰 데이터를 연산하려면 형변환이 필요해요. 예) int + short
- bool 형은 형변환에서 제외합니다.
- 작은 데이터로 변환하면 자료 상실이 되어 오류 발생해요.
- 명시적으로 변환하는 강제 형변환과, 묵시적으로 변환하는 자동 형변환이 있습니다.
- 자동형 변환을 타입 프로모션이라고 하며, 강제형변환을 캐스팅이라고 합니다.
- 타입 프로모션 

		예) double sum = 10.0f + 4.5; // float형이 double형으로
		
- 캐스팅 

		예) double sum = 1 + (double) 4; // int형에서 double형으로

#### 타입 추론, var
- - -
- Local Variable Type-Inference 
- 컴파일러가 변수의 자료형을 스스로 알아내는 것
- Java 10이후 var 로 변수를 선언하면 사용가능
- 자료형 기입하느라 에너지 쓰지 않아도 된다.
- 그 에너지로 변수 이름을 잘 만드는데 집중할 수 있다.

**주의**

- 초기화 해야 한다. 

		var n; // 초기화해야함

- null 초기화 하면 안된다. 

		var list = null; // X
		
		
- 지역변수가 아니면 작동 안함. 

	public var = "hello"; // 허용안함.
	
- 람다식은 대상형이 필요해서 형추론사용할 수 없다. (아직 잘 모르겠음)
- 배열은 명시적 형 선언이 필요함. 

		var arr = {1,2,3} ; // 배열은 안됨.
